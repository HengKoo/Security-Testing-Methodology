# Security Testing Methodology 

This repository contains three security testing methodology templates created from my own notes and practice workflows:

- `Reconnaissance-Methodology`
- `Vulnerability-Discovery-Methodology`
- `Exploitation-Methodology`

The templates are provided as professional references for structured security testing. They are intended to help organise repeatable workflows, evidence collection, validation steps, and reporting notes during authorised assessments or controlled lab work.

## Why These Files Exist

These documents are kept here as a personal methodology library and portfolio reference. They show how I approach security testing in phases:

1. Reconnaissance and information gathering.
2. Vulnerability discovery and validation.
3. Controlled proof-of-concept and impact demonstration.

The goal is to maintain clear, reusable templates that support disciplined testing, accurate documentation, and responsible reporting.

## Authorised Use Only

These templates are for authorised and controlled use only.

Do not use these workflows, commands, notes, or testing ideas against systems, networks, applications, accounts, or services unless you have explicit permission from the owner and are operating within an agreed scope.

The content is intended for:

- Authorised security assessments.
- Controlled lab environments.
- Personal learning and methodology development.
- Professional portfolio demonstration.
